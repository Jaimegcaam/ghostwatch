# Configuration

**Most installs need nothing here.** `npm run docker:setup` (or `docker:init`) sets sensible defaults.

Open this file when you need email, custom domains, multiple regions, or Kubernetes mapping.

---

## Required for production

| Variable | What to set |
| --- | --- |
| `NEXTAUTH_URL` | Full URL users open (e.g. `https://monitor.company.com`) |
| `APP_HOST` | Hostname only (e.g. `monitor.company.com`) |
| `AUTH_SECRET` | Auto-generated by setup |
| `CRON_SECRET` | Auto-generated by setup |

Docker Compose creates `DATABASE_URL` automatically.

---

## Access control

Invite-only after the owner exists.

| Variable | What to set |
| --- | --- |
| `OWNER_EMAIL` | Optional — lock the first account to your email |

---

## Status pages

Built-in URL: `https://<APP_HOST>/s/<slug>`

**Custom domain:** Dashboard → Status Pages → Custom domains → add hostname → CNAME to `APP_HOST` → **Check DNS**.

| Variable | What to set |
| --- | --- |
| `STATUS_CNAME_TARGET` | Shown in DNS instructions (defaults to `APP_HOST`) |

---

## Monitoring regions

**Single server:** nothing to configure — checks run from your server.

**Multi-region:** see [Docker](deploy/docker-multi-region.md) or [Kubernetes](deploy/kubernetes-multi-region.md) guides.

| Variable | Where |
| --- | --- |
| `PROBE_ENDPOINTS` | Hub — `id\|url\|Label` per worker |
| `PROBE_WORKER=true` | Each worker |
| `PROBE_REGION` | Each worker — same id as in `PROBE_ENDPOINTS` |

---

## Email (optional)

```bash
RESEND_API_KEY=re_...
FROM_EMAIL="Ghostwatch <alerts@yourdomain.com>"
```

Restart after editing `.env`.

---

## OAuth (optional)

`GOOGLE_CLIENT_*` and `GITHUB_*` — leave blank for password login only.

---

## Helm mapping

| `.env` | Helm |
| --- | --- |
| `NEXTAUTH_URL` | `app.nextauthUrl` |
| `APP_HOST` | `app.host` |
| `PROBE_ENDPOINTS` | `app.probeEndpoints` |
| `PROBE_WORKER` | `app.probeWorker` |
| `AUTH_SECRET` | `secrets.authSecret` |
| `CRON_SECRET` | `secrets.cronSecret` |

Full list: [`charts/ghostwatch/values.yaml`](../charts/ghostwatch/values.yaml)
